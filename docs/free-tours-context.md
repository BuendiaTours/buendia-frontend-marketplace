# Free Tours — Context for Backoffice Implementation

## What is a Free Tour

A **free tour** is a curated editorial product that groups multiple free-tour-type activities under a single experience with its own title, description, itinerary (stages), image gallery, and URL slug. Activities are assigned with **priority**: the system offers the highest-priority activity that has availability, falling back to the next one if unavailable.

Free tours are independent editorial products — they don't replace the activities they contain, but wrap them in a richer presentation layer for discovery and booking.

## Domain Model

### Aggregate: FreeTour

| Field              | Type                                    | Mutable   | Notes                                             |
| ------------------ | --------------------------------------- | --------- | ------------------------------------------------- |
| `id`               | UUID                                    | No        |                                                   |
| `categoryIds`      | UUID[]                                  | Yes       | Category references (same as activity categories) |
| `descriptionFull`  | string                                  | Yes       | Full HTML/rich description                        |
| `descriptionShort` | string \| null                          | Yes       | Summary text (nullable)                           |
| `destinationIds`   | UUID[]                                  | Yes       | Location references                               |
| `entries`          | Entry[]                                 | Yes       | Child entities: linked activities with priority   |
| `images`           | EntityImage[]                           | Read-only | Resolved from multimedia system                   |
| `slug`             | string (unique)                         | Yes       | URL-friendly identifier                           |
| `stages`           | Stage[]                                 | Yes       | Child entities: itinerary steps                   |
| `status`           | `DRAFT` \| `PUBLISHED` \| `UNPUBLISHED` | Yes       | Publication lifecycle                             |
| `title`            | string                                  | Yes       | Display title                                     |

### Child Entity: Entry

An entry links a FREE_TOUR-type activity to the free tour with a priority.

| Field        | Type   | Notes                                                        |
| ------------ | ------ | ------------------------------------------------------------ |
| `id`         | UUID   | Entity ID (generated by frontend)                            |
| `activityId` | UUID   | Must reference an activity of kind `FREE_TOUR`               |
| `priority`   | number | Lower = higher priority. Must be unique within the free tour |

### Child Entity: Stage

A stage is a step in the itinerary — either a visit (EXPERIENCE) or a transit (TRANSFER).

| Field         | Type                                              | Notes                             |
| ------------- | ------------------------------------------------- | --------------------------------- |
| `id`          | UUID                                              | Entity ID (generated by frontend) |
| `coords`      | `{ latitude, longitude }` \| null                 | Required if kind = EXPERIENCE     |
| `description` | string \| null                                    | Optional                          |
| `duration`    | number                                            | Minutes                           |
| `kind`        | `TRANSFER` \| `EXPERIENCE`                        | Determines required fields        |
| `name`        | string \| null                                    | Required if kind = EXPERIENCE     |
| `order`       | number                                            | Sequential position in itinerary  |
| `relevance`   | `HIGH` \| `MEDIUM` \| `LOW` \| `NONE`             | How important this stage is       |
| `requirement` | `REQUIRED` \| `OPTIONAL` \| `SUGGESTED` \| `NONE` | Whether this stage is mandatory   |

### Status Lifecycle

```
DRAFT → PUBLISHED → UNPUBLISHED → PUBLISHED (toggle)
                                → DRAFT
```

- Created as `DRAFT`
- Can only publish if the free tour has at least one entry
- Can unpublish back. Can return to DRAFT from UNPUBLISHED

## API Contract

Base path: `/free-tours`

### CRUD

| Method   | Path              | Description      | Body                                                                                             |
| -------- | ----------------- | ---------------- | ------------------------------------------------------------------------------------------------ |
| `POST`   | `/free-tours`     | Create           | `{ id, title, slug, descriptionShort, descriptionFull, categoryIds?, destinationIds? }`          |
| `PATCH`  | `/free-tours/:id` | Update (partial) | `{ title?, slug?, descriptionShort?, descriptionFull?, status?, categoryIds?, destinationIds? }` |
| `DELETE` | `/free-tours/:id` | Delete           | —                                                                                                |

### Queries

| Method | Path              | Returns                                                  |
| ------ | ----------------- | -------------------------------------------------------- |
| `GET`  | `/free-tours`     | `CriteriaResult<FreeTour>` — paginated list with filters |
| `GET`  | `/free-tours/:id` | `FreeTour` — full projection                             |

### Sub-Resources: Entries

| Method   | Path                               | Body                           | Notes                           |
| -------- | ---------------------------------- | ------------------------------ | ------------------------------- |
| `POST`   | `/free-tours/:id/entries`          | `{ id, activityId, priority }` | Activity must be type FREE_TOUR |
| `DELETE` | `/free-tours/:id/entries/:entryId` | —                              |                                 |

### Sub-Resources: Stages

| Method   | Path                              | Body                                                                                  | Notes                             |
| -------- | --------------------------------- | ------------------------------------------------------------------------------------- | --------------------------------- |
| `POST`   | `/free-tours/:id/stages`          | `{ id, kind, duration, order, relevance, requirement, name?, description?, coords? }` | EXPERIENCE requires name + coords |
| `DELETE` | `/free-tours/:id/stages/:stageId` | —                                                                                     |                                   |

### Criteria Filters

The GET `/free-tours` endpoint accepts these query params:

| Param         | Type                                    | Description                                  |
| ------------- | --------------------------------------- | -------------------------------------------- |
| `id`          | string                                  | Filter by ID                                 |
| `slug`        | string                                  | Filter by exact slug                         |
| `status`      | `DRAFT` \| `PUBLISHED` \| `UNPUBLISHED` | Filter by status                             |
| `title`       | string                                  | Filter by title                              |
| `search_text` | string                                  | Full-text search on title + descriptionShort |
| `skip`        | number                                  | Pagination offset                            |
| `limit`       | number                                  | Page size                                    |
| `sort`        | `title` \| `createdAt`                  | Sort field                                   |
| `operator`    | `AND` \| `OR`                           | Combine filters                              |
| `order`       | `ASC` \| `DESC`                         | Sort direction                               |

**Note**: The criteria DTO uses `search_text` (snake_case) — this is a known inconsistency with the rest of the API which uses camelCase.

## Backoffice Implementation Notes

### List Page (`/backoffice/free-tours`)

Suggested filters:

- **Text search**: `title` or `search_text`
- **Status**: Dropdown with DRAFT/PUBLISHED/UNPUBLISHED
- **Sort**: title (ASC/DESC), createdAt (ASC/DESC)

Table columns: title, status (badge), entries count, stages count, slug.

### Create Page (`/backoffice/free-tours/create`)

Form fields:

- `title` (text, required)
- `slug` (text, required — auto-generate from title)
- `descriptionShort` (text, required)
- `descriptionFull` (rich text / textarea, required)
- `categoryIds` (multi-select async search, optional)
- `destinationIds` (multi-select async search, optional)

Entries and stages are managed in the edit page.

### Edit Page (`/backoffice/free-tours/:id/edit`)

This is the complex page with multiple sections:

1. **Basic fields section**: title, slug, descriptionShort, descriptionFull, categoryIds, destinationIds
2. **Status section**: Publish/Unpublish toggle (status managed via PATCH with `{ status: 'PUBLISHED' }`)
3. **Entries section** (linked activities):
   - Table: activityId (resolve to activity name via search), priority
   - Add: async search for activities (filter by kind=FREE_TOUR), input priority number
   - Remove: delete button per entry
   - **Important**: Only activities of type `FREE_TOUR` can be added. Priority must be unique.
   - Frontend must generate UUID for the entry `id` field
4. **Stages section** (itinerary):
   - Ordered list: kind badge, name, duration, order
   - Add form: kind selector → if EXPERIENCE: name + coords required; duration, order, relevance, requirement always required
   - Remove: delete button per stage
   - Frontend must generate UUID for the stage `id` field
   - Consider drag-and-drop reorder (would need remove + re-add with new order values)
5. **Images section**: Media gallery (if media picker component exists)

### Status (Publish/Unpublish)

Status is managed through PATCH (not dedicated endpoints):

```typescript
// Publish
await FREE_TOUR_REQUEST.update(fetch, id, { status: 'PUBLISHED' });

// Unpublish
await FREE_TOUR_REQUEST.update(fetch, id, { status: 'UNPUBLISHED' });
```

**Publish validation**: The backend rejects publish if the free tour has no entries. Handle error code `FREE_TOUR_CANNOT_PUBLISH_WITHOUT_ENTRIES`.

### Validation Rules

| Rule                                    | UX Impact                                       |
| --------------------------------------- | ----------------------------------------------- |
| `title` required                        | Form validation                                 |
| `slug` required, unique system-wide     | Show error if slug taken                        |
| `descriptionShort` optional (nullable)  | Can be omitted or set to null                   |
| `descriptionFull` required              | Form validation                                 |
| EXPERIENCE stage requires name + coords | Conditional form fields based on kind selection |
| Entry priority must be unique           | Validate before submit, show error              |
| Entry activity must be type FREE_TOUR   | Async search should filter by kind              |
| Cannot publish without entries          | Show error message on publish attempt           |

### Error Codes to Handle

| Code                                                  | User Message                                          |
| ----------------------------------------------------- | ----------------------------------------------------- |
| `FREE_TOUR_ALREADY_EXISTS`                            | A free tour with this ID already exists               |
| `FREE_TOUR_NOT_FOUND`                                 | Free tour not found                                   |
| `FREE_TOUR_CANNOT_PUBLISH_WITHOUT_ENTRIES`            | Cannot publish: add at least one activity entry first |
| `FREE_TOUR_ENTRY_ALREADY_EXISTS`                      | This entry already exists                             |
| `FREE_TOUR_ENTRY_DUPLICATE_PRIORITY`                  | An entry with this priority already exists            |
| `FREE_TOUR_ENTRY_NOT_FOUND`                           | Entry not found                                       |
| `FREE_TOUR_ACTIVITY_NOT_FREE_TOUR`                    | The selected activity is not of type "Free Tour"      |
| `FREE_TOUR_STAGE_EXPERIENCE_REQUIRES_NAME_AND_COORDS` | Experience stages require a name and coordinates      |

## Relationships with Other Resources

| Resource                   | Relationship                   | How it works                                                                                         |
| -------------------------- | ------------------------------ | ---------------------------------------------------------------------------------------------------- |
| **Activity**               | Via entries (priority-ordered) | Only activities with `kind: FREE_TOUR` can be linked. Each activity can belong to only one free tour |
| **Category**               | Direct UUID array              | `categoryIds` stored directly on the aggregate. Updated via PATCH                                    |
| **Location (Destination)** | Direct UUID array              | `destinationIds` stored directly on the aggregate. Updated via PATCH                                 |
| **Media/Images**           | Via multimedia system          | Gallery managed externally. Projection includes resolved `images[]` with variants                    |
| **Slug**                   | Auto-managed                   | Created/updated/deleted events are consumed by the Slug context for URL management                   |

## Key Behavioral Notes

1. **CQRS eventual consistency**: After create/update/delete, the read projection takes ~500ms to propagate. Use `redirectDelayMs` in action factories.
2. **UUID generation**: The frontend must generate UUIDs for `id` fields when adding entries and stages. Use `crypto.randomUUID()`.
3. **Entries are the core value**: A free tour without entries is just a draft shell. The business value comes from linking prioritized activities.
4. **Stage ordering**: Stages have an `order` field that determines their position in the itinerary. The frontend should present them sorted by order.
5. **Conditional stage fields**: When `kind = EXPERIENCE`, `name` and `coords` become required. The form should show/hide these fields based on the kind selection.
6. **Priority uniqueness**: Entry priorities must be unique. The form should validate this client-side before submitting.
7. **Activity search for entries**: When adding an entry, the activity search should filter by `kind: FREE_TOUR` to only show eligible activities. Use the activity criteria endpoint with the appropriate filter.
8. **Slug uniqueness**: Validated across all catalog entities (activities, attractions, categories, locations, distributives, free tours) by the Slug context.
